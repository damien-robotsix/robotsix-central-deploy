Add `PUT /chat/github/repos/{owner}/{repo}/vulnerability-alerts` to enable a repo's Dependency graph / Dependabot alerts, and call it automatically at the end of `POST /chat/github/repos` (repo creation) — without it, CI checks that depend on the dependency graph (e.g. `dependency-review-action`) fail on every PR until a human manually flips Settings > Security > Dependency graph, which was blocking a chain of `robotsix-board` mill tickets in `fixing_ci`.
