Follow-up to the virtual-component ServiceRecord leak fix: component configs persisted by the original (pre-`is_virtual`) seeding still loaded back with `is_virtual=False`, so their stale `ServiceRecord`s survived a restart. The seeding loop now also matches persisted config ids against the current `virtual_components` list and backfills `is_virtual=True` on them, so already-leaked `deploy`/`langfuse` rows actually clear on the next restart.
