Fix a logging bug where every application log record silently failed to emit and was replaced with a generic "--- Logging error ---" block instead of the actual message. `LOGGING_CONFIG`'s JSON formatter listed its structlog processors (`foreign_pre_chain`/`processors`) as dotted-path strings, but `dictConfig`'s `"()"` resolution only resolves the top-level formatter class, not nested keys — the strings were passed through unresolved, and `ProcessorFormatter.format()` tried to call them, raising `TypeError: 'str' object is not callable` on every single log call.
